# 🤖 ربات مدیریت گروه مودو (Modo)

یک ربات قدرتمند و چندگروهی برای مدیریت گروه‌های تلگرام با داشبورد وب، تخصصی در **مدیریت دسترسی موضوعات فروم** و مجوزهای دقیق کاربران.

ساخته شده با **Aiogram 3.x** (async)، **FastAPI**، **PostgreSQL**، **Redis** و **Docker**.

## ✨ امکانات

### 🔒 مدیریت دسترسی موضوعات فروم (Topic ACL)
- محدود کردن موضوعات خاص فروم به کاربران لیست سفید
- مدیریت لیست سفید کاربران برای هر موضوع
- حذف خودکار پیام‌های کاربران غیرمجاز

### 🛡️ ابزارهای مدیریت گروه
- **ضد اسپم**: محدودیت نرخ پیام با تنظیمات قابل تنظیم
- **ضد لینک**: حذف خودکار پیام‌های حاوی URL
- **فیلتر کلمات**: مسدود کردن خودکار کلمات ممنوعه
- **سیستم اخطار**: `/warn`، `/warnings`، `/resetwarns`
- **میوت/بن/کیک**: اقدامات سریع مدیریتی
- **کپچا**: چالش ریاضی برای اعضای جدید

### 📝 پیام‌های سفارشی
- پیام خوش‌آمدگویی
- پیام خداحافظی
- قالب‌های اخطار
- چالش‌های کپچا
- همه از طریق داشبورد وب قابل ویرایش

### 🌐 داشبورد وب
- ورود با تلگرام (بدون نیاز به رمز عبور)
- مدیریت تمام گروه‌ها از یک مکان
- لاگ فعالیت‌ها به صورت بلادرنگ
- REST API برای اتوماسیون

## 🚀 راهنمای راه‌اندازی برای صاحبان ربات

### پیش‌نیازها

- سرور با Docker و Docker Compose نصب‌شده
- دامنه‌ای که به IP سرور شما اشاره می‌کند
- توکن ربات تلگرام از [@BotFather](https://t.me/BotFather)

### مرحله ۱: دریافت کد

```bash
git clone https://github.com/netadminplus/modo.git
cd modo
```

### مرحله ۲: تنظیمات محیط

```bash
cp .env.example .env
nano .env
```

فایل `.env` را با مقادیر خود ویرایش کنید:

```ini
# تنظیمات ربات
BOT_TOKEN=123456789:AAF...          # از @BotFather
BOT_USERNAME=your_bot_username       # بدون @
ADMIN_IDS=123456789                  # شناسه کاربری تلگرام شما (از @userinfobot)

# پایگاه داده
POSTGRES_USER=botadmin
POSTGRES_PASSWORD=رمز_امن_اینجا
POSTGRES_DB=telegram_bot
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
DATABASE_URL=postgresql+asyncpg://botadmin:رمز_امن_اینجا@postgres:5432/telegram_bot

# Redis
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=رمز_ردیس_اینجا
REDIS_URL=redis://:رمز_ردیس_اینجا@redis:6379/0

# داشبورد وب
SECRET_KEY=یک_رشته_تصادفی_۶۴_کاراکتری_اینجا_تولید_کنید
DOMAIN=yourdomain.com
WEB_PORT=8000

# سایر
DEBUG=false
LOG_LEVEL=INFO
TIMEZONE=UTC
```

**دریافت شناسه کاربری تلگرام:** به [@userinfobot](https://t.me/userinfobot) پیام دهید

**تولید SECRET_KEY:** دستور زیر را اجرا کنید:
```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### مرحله ۳: تنظیم DNS

دامنه خود را به سرور متصل کنید:

```
Type: A
Name: @ (یا yourdomain.com)
Value: IP_SERVER_SHOMA
TTL: 300
```

### مرحله ۴: گواهی SSL (فقط بار اول)

```bash
# اجرای nginx
docker compose up -d nginx

# دریافت گواهی SSL
docker compose run --rm certbot

# راه‌اندازی مجدد nginx با SSL
docker compose restart nginx
```

### مرحله ۵: شروع تمام سرویس‌ها

```bash
docker compose up -d
```

### مرحله ۶: بررسی صحت عملکرد

```bash
# بررسی اجرای کانتینرها
docker compose ps

# مشاهده لاگ ربات
docker compose logs -f app

# بررسی endpoint سلامت
curl https://yourdomain.com/health
```

پاسخ مورد انتظار: `{"status":"ok"}`

### مرحله ۷: دسترسی به داشبورد وب

آدرس `https://yourdomain.com` را در مرورگر باز کنید و با تلگرام وارد شوید.

## 👥 راهنمای استفاده برای مدیران گروه

### افزودن ربات به گروه

1. **یافتن ربات** در تلگرام (جستجوی نام کاربری ربات)
2. **افزودن به گروه**: ورود به اطلاعات گروه → افزودن عضو → انتخاب ربات
3. **مدیر کردن ربات**: اطلاعات گروه → ویرایش → مدیران → افزودن مدیر → انتخاب ربات

### دسترسی‌های مورد نیاز ربات

ربات به این دسترسی‌های مدیریتی نیاز دارد:

- ✅ حذف پیام‌ها
- ✅ بن کردن کاربران
- ✅ محدود کردن اعضا (میوت)
- ✅ مدیریت موضوعات (برای گروه‌های فروم)
- ✅ سنجاق کردن پیام‌ها (اختیاری)

### ثبت گروه

پس از اینکه ربات را در گروه خود مدیر کردید:

1. چت گروه را باز کنید
2. ارسال: `/register`
3. ربات پاسخ می‌دهد: `✅ Group registered!`
4. گروه شما اکنون در داشبورد وب نمایش داده می‌شود

### دسترسی به تنظیمات گروه

1. به آدرس `https://yourdomain.com` بروید
2. روی "Login with Telegram" کلیک کنید
3. به ربات اجازه دسترسی دهید
4. گروه خود را از داشبورد انتخاب کنید

## 📖 سناریوهای کاربردی

### سناریو ۱: راه‌اندازی فروم پشتیبانی

**وضعیت:** شما یک گروه پشتیبانی مشتری با موضوعات مختلف برای محصولات مختلف دارید.

**راه‌اندازی:**
```
1. فعال کردن حالت فروم در گروه تلگرام
2. ایجاد موضوعات: "محصول الف"، "محصول ب"، "عمومی"
3. افزودن ربات و مدیر کردن آن
4. ارسال /register در گروه
5. در داشبورد، به تب Topics بروید
6. موضوع "محصول الف" را محدود کنید
7. فقط تیم پشتیبانی محصول الف را در لیست سفید قرار دهید
```

**دستورات:**
```
/restrict_topic          # قفل کردن موضوع فعلی
/allow_user 123456789    # اجازه به کاربر خاص
/topic_users             # مشاهده کاربران مجاز
```

**نتیجه:** فقط کارکنان پشتیبانی در لیست سفید می‌توانند در موضوع محصول الف پست بگذارند. مشتریان می‌توانند بخوانند اما اسپم نکنند.

---

### سناریو ۲: جلوگیری از Flood و اسپم

**وضعیت:** گروه شما با پیام‌ها و لینک‌ها پر می‌شود.

**تنظیمات داشبورد:**
1. رفتن به Group Settings → Moderation
2. فعال کردن "Anti-Flood"
3. تنظیم threshold: ۵ پیام در ۱۰ ثانیه
4. Action: Mute به مدت ۵ دقیقه
5. فعال کردن "Anti-Link"
6. افزودن کلمات ممنوعه: "crypto"، "investment"، "free money"

**نتیجه:** کاربرانی که اسپم می‌کنند یا لینک می‌فرستند به صورت خودکار میوت می‌شوند. کلمات ممنوعه خودکار حذف می‌شوند.

---

### سناریو ۳: سیستم اخطار برای متخلفان

**وضعیت:** می‌خواهید یک سیستم اخطار عادلانه قبل از بن کردن داشته باشید.

**تنظیمات داشبورد:**
1. Group Settings → Moderation
2. تنظیم Max Warnings: ۳
3. Auto-action: Ban

**در گروه:**
```
/warn @username Spamming
/warn @username          # ریپلای به پیام
/warnings @username      # بررسی تعداد اخطار
/resetwarns @username    # ریست کردن اخطارها
```

**روند:**
- اخطار ۱: کاربر اخطار می‌گیرد
- اخطار ۲: کاربر اخطار می‌گیرد
- اخطار ۳: کاربر به صورت خودکار بن می‌شود

---

### سناریو ۴: تأیید اعضای جدید

**وضعیت:** ربات‌ها و اسپمرها به گروه شما می‌پیوندند.

**تنظیمات داشبورد:**
1. Group Settings → Moderation
2. فعال کردن "Captcha"
3. سفارشی کردن پیام خوش‌آمدگویی: `Welcome {user_mention}! Solve: {a} + {b} = ?`

**اتفاقی که می‌افتد:**
1. عضو جدید می‌پیوندد
2. ربات او را محدود می‌کند (فقط خواندنی)
3. ربات کپچا می‌فرستد: "Solve: 5 + 3 = ?"
4. کاربر پاسخ می‌دهد: "8"
5. ربات او را از میوت خارج کرده و خوش‌آمد می‌گوید

**نتیجه:** ربات‌ها در کپچا شکست خورده و خارج می‌شوند. انسان‌های واقعی به راحتی عبور می‌کنند.

---

### سناریو ۵: مدیریت چندگروهی

**وضعیت:** شما ۱۰ گروه مختلف را مدیریت می‌کنید.

**امکانات داشبورد:**
1. یک ورود تمام گروه‌های شما را نشان می‌دهد
2. جابجایی آنی بین گروه‌ها
3. کپی کردن تنظیمات از یک گروه به گروه دیگر
4. مشاهده لاگ فعالیت‌ها در تمام گروه‌ها
5. اقدامات گروهی (به زودی)

**مزیت:** نیازی به بازدید از هر گروه به صورت جداگانه نیست. همه چیز را از یک داشبورد مدیریت کنید.

---

### سناریو ۶: محدودیت موقت

**وضعیت:** گروه شما تحت حمله است، نیاز به قفل اضطراری دارید.

**اقدامات سریع:**
```
1. داشبورد → Group Settings
2. فعال کردن "Restrict All Members"
3. فقط مدیران می‌توانند پست بگذارند
4. حمله بلافاصله متوقف می‌شود
5. پس از ایمن بودن غیرفعال کنید
```

**جایگزین:** محدود کردن موضوعات خاص به جای کل گروه.

---

### سناریو ۷: قالب‌های پیام سفارشی

**وضعیت:** می‌خواهید پیام‌های شخصی‌سازی شده با صدای برند خود داشته باشید.

**داشبورد:**
1. Group Settings → Message Templates
2. ویرایش هر قالب:
   - پیام خوش‌آمدگویی
   - پیام اخطار
   - چالش کپچا
   - پیام خداحافظی

**متغیرهای موجود:**
- `{user_mention}` - لینک @username
- `{user_name}` - نام نمایشی
- `{group_title}` - نام گروه
- `{count}` - تعداد اخطار
- `{max}` - حداکثر اخطار
- `{reason}` - دلیل اخطار
- `{duration}` - مدت میوت

**مثال:**
```
Welcome {user_mention} to {group_title}! 
لطفاً قوانین را مطالعه کنید و لذت ببرید.
```

## 🛠️ توسعه

### راه‌اندازی توسعه محلی

```bash
# کلون و راه‌اندازی
git clone https://github.com/netadminplus/modo.git
cd modo
python -m venv venv
source venv/bin/activate  # لینوکس/Mac
pip install -r requirements.txt

# شروع پایگاه‌های داده
docker compose up -d postgres redis

# اجرای مایگریشن‌ها
alembic upgrade head

# شروع ربات (ترمینال ۱)
python -m bot.main polling

# شروع داشبورد وب (ترمینال ۲)
uvicorn web.app:app --reload --port 8000
```

### محیط توسعه

```ini
DEBUG=true
LOG_LEVEL=DEBUG
DOMAIN=localhost
```

### اجرای مایگریشن‌ها

```bash
# ایجاد مایگریشن جدید
alembic revision --autogenerate -m "add_feature"

# اعمال مایگریشن‌ها
alembic upgrade head

# بازگشت
alembic downgrade -1
```

## 🔐 امنیت

- تمام رمزها در `.env` (هرگز کامیت نمی‌شود)
- سشن‌های داشبورد در Redis (قابل ابطال)
- ورود تلگرام با HMAC-SHA256 تأیید می‌شود
- Nginx نرخ ورود و API را محدود می‌کند
- کاربر Docker غیر-root
- پایگاه داده فقط به localhost متصل است
- HSTS + TLS 1.2/1.3 فقط

## 🎯 مرجع سریع دستورات

### دستورات ربات

| دستور | توضیح |
|-------|-------|
| `/register` | ثبت گروه فعلی |
| `/settings` | باز کردن داشبورد وب |
| `/warn` | اخطار به کاربر |
| `/warnings` | بررسی اخطارهای کاربر |
| `/resetwarns` | ریست اخطارهای کاربر |
| `/mute` | میوت موقت کاربر |
| `/ban` | بن کردن کاربر |
| `/kick` | کیك کردن کاربر |
| `/restrict_topic` | قفل کردن موضوع فعلی |
| `/unrestrict_topic` | باز کردن قفل موضوع |
| `/allow_user` | افزودن کاربر به لیست سفید موضوع |
| `/deny_user` | حذف از لیست سفید |
| `/topic_users` | لیست کاربران مجاز |
